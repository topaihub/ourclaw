# planning

`ourclaw/docs/planning/` 主要用于阶段恢复、任务背景和历史分析，不是 active spec 的主入口。

## 当前优先阅读

- `session-resume.md`
  - 当前阶段结论、最近完成项、下一步续作入口
- `restart-handoff.md`
  - 会话/IDE 中断后的超短恢复入口
- `nullclaw-gap-analysis.md`
  - 对标 `nullclaw` 时的差距分析基线

## 历史参考

- `../backup/planning/next-session-handoff-2026-03-13.md`
- `../backup/planning/full-business-gap-tasks.md`

## 怎么用这个目录

- 要继续当前主线开发：先看 `session-resume.md`
- 要恢复一次中断现场：看 `restart-handoff.md`
- 要理解更老的任务背景：看 `backup/planning/*`

## 一句结论

`planning/` 是恢复和背景目录；
真正的主入口仍然是 `docs/specs/reference-aligned-ourclaw/`。
